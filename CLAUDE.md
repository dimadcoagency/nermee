## 🧭 PROJECT IDENTITY

**App Name:** Nearmee
**Tagline:** "Services at your doorstep"
**Bisaya Tagline:** "Serbisyo hapit ra nimo"
**What it is:** A local services marketplace PWA connecting customers with nearby service providers (plumbers, drivers, cooks, cleaners, tutors, etc.)
**Who built it:** Dimad, founder of MADINNO (Making A Dynamic Innovation), a digital agency and AI-powered software development operation based in the Philippines.
**Launch markets:** Bayawan City + Dumaguete City, Negros Oriental, Philippines
**Business model:** Freemium marketplace — free for customers, merchants pay for premium features

---

## 🎯 STRATEGIC CONTEXT (read this — it shapes every technical decision)

### Why provincial Philippines?
- Zero competitors. GoodWork.ph is Manila-only. Gawin.ph is dead. MyKuya is Manila-centric.
- Smaller cities = faster network effects. 30 merchants covers Bayawan's entire commercial district.
- 270K+ combined population across Bayawan (126K) + Dumaguete (142K).
- Dumaguete is a UNESCO Creative City with 4 universities = built-in early adopters.

### Why no payment processing at launch?
- Reduces legal requirements (no need for full DTI/PayMongo integration on day 1)
- Customers pay merchants directly (cash, GCash person-to-person, bank transfer)
- This is how Grab, GoodWork, and Gawin all started
- PayMongo integration comes in Phase 5 (Month 5-8) after market validation

### Why PWA, not native app?
- Instant deployment via Vercel — no Play Store approval wait
- Users install from browser to home screen — looks and feels like a native app
- One codebase serves all Android devices (our target market)
- Play Store listing comes later (Month 9-12) via TWA or Capacitor wrapper

### Revenue model (don't build payment features yet — just the data structures)
- Phase 1 (Month 3): Subscriptions (Starter FREE, Pro ₱299/mo, Business ₱899/mo)
- Phase 1 (Month 3): Boost listings (₱29-149), Verified badges (₱99)
- Phase 2 (Month 6): 5% booking commission + ₱15 convenience fee via PayMongo
- Phase 3 (Year 2): Local business ads, deal campaigns

## 🏗️ TECH STACK

| Layer | Technology | Why |
|-------|-----------|-----|
| Framework | **Next.js 14+ (App Router)** | SSR, file-based routing, API routes, PWA-ready |
| Styling | **Tailwind CSS** | Utility-first, fast iteration, mobile-first |
| Database | **Supabase (PostgreSQL)** | Free tier (500MB), built-in auth, real-time, Row Level Security |
| Auth | **Supabase Auth (Phone OTP)** | Filipinos identify by phone number, not email |
| Storage | **Supabase Storage** | Merchant photos, service images, KYC documents |
| Hosting | **Vercel** | Free tier for testing, $20/mo Pro for commercial |
| Maps | **Leaflet + OpenStreetMap** | Free, no API key needed (Google Maps as upgrade later) |
| State | **React Context + Supabase real-time** | Simple, no Redux needed for MVP |
| PWA | **next-pwa** | Service worker, offline support, installable |

### What we are NOT using (and why)
- **No Redux/Zustand** — Context API is enough for MVP. Don't over-engineer state.
- **No Prisma** — Use Supabase client directly. One less abstraction layer.
- **No NextAuth** — Supabase Auth handles everything including phone OTP.
- **No Stripe/PayMongo yet** — Direct payment model at launch. Payment integration is Phase 5.
- **No React Native/Flutter** — PWA first. Native wrapper comes in Phase 6.

## 🗄️ DATABASE SCHEMA

This is the source of truth. Copy this to `supabase/migrations/001_initial_schema.sql`.

```sql
-- ============================================
-- NEARMEE DATABASE SCHEMA v1.0
-- Supabase (PostgreSQL)
-- ============================================

-- Enable necessary extensions
CREATE EXTENSION IF NOT EXISTS "uuid-ossp";
CREATE EXTENSION IF NOT EXISTS "postgis";  -- For location queries later

-- ============================================
-- ENUMS
-- ============================================
CREATE TYPE user_role AS ENUM ('customer', 'merchant');
CREATE TYPE merchant_tier AS ENUM ('free', 'pro', 'business');
CREATE TYPE booking_status AS ENUM ('pending', 'confirmed', 'in_progress', 'completed', 'cancelled', 'no_show');
CREATE TYPE service_status AS ENUM ('active', 'paused', 'deleted');

-- ============================================
-- PROFILES (extends Supabase auth.users)
-- ============================================
CREATE TABLE profiles (
    id UUID PRIMARY KEY REFERENCES auth.users(id) ON DELETE CASCADE,
    phone TEXT,
    full_name TEXT NOT NULL,
    avatar_url TEXT,
    role user_role DEFAULT 'customer',
    city TEXT DEFAULT 'Bayawan City',
    created_at TIMESTAMPTZ DEFAULT NOW(),
    updated_at TIMESTAMPTZ DEFAULT NOW()
);

-- ============================================
-- MERCHANTS
-- ============================================
CREATE TABLE merchants (
    id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
    user_id UUID NOT NULL REFERENCES profiles(id) ON DELETE CASCADE,
    business_name TEXT NOT NULL,
    phone TEXT NOT NULL,
    service_area TEXT NOT NULL,       -- e.g. "Bayawan City, Poblacion"
    category TEXT NOT NULL,           -- Primary category
    bio TEXT,
    tier merchant_tier DEFAULT 'free',
    is_verified BOOLEAN DEFAULT FALSE,
    rating_avg DECIMAL(2,1) DEFAULT 0.0,
    rating_count INTEGER DEFAULT 0,
    is_active BOOLEAN DEFAULT TRUE,
    created_at TIMESTAMPTZ DEFAULT NOW(),
    updated_at TIMESTAMPTZ DEFAULT NOW(),
    UNIQUE(user_id)                   -- One merchant profile per user
);

-- ============================================
-- SERVICES
-- ============================================
CREATE TABLE services (
    id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
    merchant_id UUID NOT NULL REFERENCES merchants(id) ON DELETE CASCADE,
    title TEXT NOT NULL,
    description TEXT,
    category TEXT NOT NULL,
    price DECIMAL(10,2) NOT NULL,
    price_unit TEXT DEFAULT 'per visit', -- per visit, per hour, per km, per kilo, per meal, per session, per task
    image_url TEXT,
    status service_status DEFAULT 'active',
    is_boosted BOOLEAN DEFAULT FALSE,
    boost_expires_at TIMESTAMPTZ,
    city TEXT NOT NULL DEFAULT 'Bayawan City',
    created_at TIMESTAMPTZ DEFAULT NOW(),
    updated_at TIMESTAMPTZ DEFAULT NOW()
);

-- ============================================
-- AVAILABILITY (time slots per service)
-- ============================================
CREATE TABLE availability (
    id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
    service_id UUID NOT NULL REFERENCES services(id) ON DELETE CASCADE,
    day_of_week INTEGER NOT NULL CHECK (day_of_week BETWEEN 0 AND 6), -- 0=Sunday
    time_slot TEXT NOT NULL,           -- e.g. "9:00 AM", "2:00 PM"
    is_available BOOLEAN DEFAULT TRUE
);

-- ============================================
-- BOOKINGS
-- ============================================
CREATE TABLE bookings (
    id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
    customer_id UUID NOT NULL REFERENCES profiles(id),
    merchant_id UUID NOT NULL REFERENCES merchants(id),
    service_id UUID NOT NULL REFERENCES services(id),
    booking_date DATE NOT NULL,
    booking_time TEXT NOT NULL,
    notes TEXT,
    status booking_status DEFAULT 'pending',
    total_price DECIMAL(10,2),
    customer_rating INTEGER CHECK (customer_rating BETWEEN 1 AND 5),
    customer_review TEXT,
    rated_at TIMESTAMPTZ,
    created_at TIMESTAMPTZ DEFAULT NOW(),
    updated_at TIMESTAMPTZ DEFAULT NOW()
);

-- ============================================
-- SUBSCRIPTIONS (merchant tier tracking)
-- ============================================
CREATE TABLE subscriptions (
    id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
    merchant_id UUID NOT NULL REFERENCES merchants(id) ON DELETE CASCADE,
    tier merchant_tier NOT NULL DEFAULT 'free',
    started_at TIMESTAMPTZ DEFAULT NOW(),
    expires_at TIMESTAMPTZ,
    payment_method TEXT,              -- 'gcash', 'bank_transfer', later 'paymongo'
    payment_reference TEXT,           -- GCash ref number or transaction ID
    is_active BOOLEAN DEFAULT TRUE,
    created_at TIMESTAMPTZ DEFAULT NOW()
);

-- ============================================
-- BOOSTS (promoted listings)
-- ============================================
CREATE TABLE boosts (
    id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
    service_id UUID NOT NULL REFERENCES services(id) ON DELETE CASCADE,
    merchant_id UUID NOT NULL REFERENCES merchants(id),
    package TEXT NOT NULL,             -- '1d', '3d', '7d'
    price DECIMAL(10,2) NOT NULL,
    started_at TIMESTAMPTZ DEFAULT NOW(),
    expires_at TIMESTAMPTZ NOT NULL,
    payment_reference TEXT,
    is_active BOOLEAN DEFAULT TRUE,
    created_at TIMESTAMPTZ DEFAULT NOW()
);

-- ============================================
-- REPORTS (for ITA compliance — user reports)
-- ============================================
CREATE TABLE reports (
    id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
    reporter_id UUID NOT NULL REFERENCES profiles(id),
    reported_service_id UUID REFERENCES services(id),
    reported_merchant_id UUID REFERENCES merchants(id),
    reported_booking_id UUID REFERENCES bookings(id),
    reason TEXT NOT NULL,
    details TEXT,
    status TEXT DEFAULT 'pending',     -- pending, reviewed, resolved, dismissed
    admin_notes TEXT,
    created_at TIMESTAMPTZ DEFAULT NOW()
);

-- ============================================
-- INDEXES (performance)
-- ============================================
CREATE INDEX idx_services_category ON services(category);
CREATE INDEX idx_services_city ON services(city);
CREATE INDEX idx_services_status ON services(status);
CREATE INDEX idx_services_merchant ON services(merchant_id);
CREATE INDEX idx_services_boosted ON services(is_boosted) WHERE is_boosted = TRUE;
CREATE INDEX idx_bookings_customer ON bookings(customer_id);
CREATE INDEX idx_bookings_merchant ON bookings(merchant_id);
CREATE INDEX idx_bookings_status ON bookings(status);
CREATE INDEX idx_bookings_date ON bookings(booking_date);
CREATE INDEX idx_merchants_category ON merchants(category);
CREATE INDEX idx_merchants_area ON merchants(service_area);
CREATE INDEX idx_availability_service ON availability(service_id);

-- ============================================
-- FUNCTIONS
-- ============================================

-- Auto-update updated_at timestamp
CREATE OR REPLACE FUNCTION update_updated_at()
RETURNS TRIGGER AS $$
BEGIN
    NEW.updated_at = NOW();
    RETURN NEW;
END;
$$ LANGUAGE plpgsql;

CREATE TRIGGER set_profiles_updated_at BEFORE UPDATE ON profiles FOR EACH ROW EXECUTE FUNCTION update_updated_at();
CREATE TRIGGER set_merchants_updated_at BEFORE UPDATE ON merchants FOR EACH ROW EXECUTE FUNCTION update_updated_at();
CREATE TRIGGER set_services_updated_at BEFORE UPDATE ON services FOR EACH ROW EXECUTE FUNCTION update_updated_at();
CREATE TRIGGER set_bookings_updated_at BEFORE UPDATE ON bookings FOR EACH ROW EXECUTE FUNCTION update_updated_at();

-- Update merchant rating when a booking is rated
CREATE OR REPLACE FUNCTION update_merchant_rating()
RETURNS TRIGGER AS $$
BEGIN
    IF NEW.customer_rating IS NOT NULL AND OLD.customer_rating IS NULL THEN
        UPDATE merchants SET
            rating_avg = (
                SELECT ROUND(AVG(customer_rating)::numeric, 1)
                FROM bookings
                WHERE merchant_id = NEW.merchant_id AND customer_rating IS NOT NULL
            ),
            rating_count = (
                SELECT COUNT(*)
                FROM bookings
                WHERE merchant_id = NEW.merchant_id AND customer_rating IS NOT NULL
            )
        WHERE id = NEW.merchant_id;
    END IF;
    RETURN NEW;
END;
$$ LANGUAGE plpgsql;

CREATE TRIGGER on_booking_rated AFTER UPDATE ON bookings FOR EACH ROW EXECUTE FUNCTION update_merchant_rating();

-- Auto-expire boosts
CREATE OR REPLACE FUNCTION expire_boosts()
RETURNS void AS $$
BEGIN
    UPDATE boosts SET is_active = FALSE WHERE expires_at < NOW() AND is_active = TRUE;
    UPDATE services SET is_boosted = FALSE, boost_expires_at = NULL
    WHERE id IN (SELECT service_id FROM boosts WHERE expires_at < NOW() AND is_active = FALSE)
    AND is_boosted = TRUE;
END;
$$ LANGUAGE plpgsql;
```

## 🔒 ROW LEVEL SECURITY (RLS)

```sql
-- Enable RLS on all tables
ALTER TABLE profiles ENABLE ROW LEVEL SECURITY;
ALTER TABLE merchants ENABLE ROW LEVEL SECURITY;
ALTER TABLE services ENABLE ROW LEVEL SECURITY;
ALTER TABLE bookings ENABLE ROW LEVEL SECURITY;
ALTER TABLE availability ENABLE ROW LEVEL SECURITY;
ALTER TABLE subscriptions ENABLE ROW LEVEL SECURITY;
ALTER TABLE boosts ENABLE ROW LEVEL SECURITY;
ALTER TABLE reports ENABLE ROW LEVEL SECURITY;

-- PROFILES
CREATE POLICY "Public profiles are viewable by everyone" ON profiles FOR SELECT USING (true);
CREATE POLICY "Users can update their own profile" ON profiles FOR UPDATE USING (auth.uid() = id);
CREATE POLICY "Users can insert their own profile" ON profiles FOR INSERT WITH CHECK (auth.uid() = id);

-- MERCHANTS
CREATE POLICY "Merchants are viewable by everyone" ON merchants FOR SELECT USING (true);
CREATE POLICY "Users can create their own merchant profile" ON merchants FOR INSERT WITH CHECK (auth.uid() = user_id);
CREATE POLICY "Merchants can update their own profile" ON merchants FOR UPDATE USING (auth.uid() = user_id);

-- SERVICES
CREATE POLICY "Active services are viewable by everyone" ON services FOR SELECT USING (status = 'active');
CREATE POLICY "Merchants can insert their own services" ON services FOR INSERT WITH CHECK (
    merchant_id IN (SELECT id FROM merchants WHERE user_id = auth.uid())
);
CREATE POLICY "Merchants can update their own services" ON services FOR UPDATE USING (
    merchant_id IN (SELECT id FROM merchants WHERE user_id = auth.uid())
);

-- BOOKINGS
CREATE POLICY "Customers can view their own bookings" ON bookings FOR SELECT USING (
    customer_id = auth.uid() OR
    merchant_id IN (SELECT id FROM merchants WHERE user_id = auth.uid())
);
CREATE POLICY "Customers can create bookings" ON bookings FOR INSERT WITH CHECK (customer_id = auth.uid());
CREATE POLICY "Booking parties can update booking" ON bookings FOR UPDATE USING (
    customer_id = auth.uid() OR
    merchant_id IN (SELECT id FROM merchants WHERE user_id = auth.uid())
);

-- AVAILABILITY
CREATE POLICY "Availability is viewable by everyone" ON availability FOR SELECT USING (true);
CREATE POLICY "Merchants can manage their service availability" ON availability FOR ALL USING (
    service_id IN (
        SELECT s.id FROM services s
        JOIN merchants m ON s.merchant_id = m.id
        WHERE m.user_id = auth.uid()
    )
);

-- REPORTS
CREATE POLICY "Users can create reports" ON reports FOR INSERT WITH CHECK (reporter_id = auth.uid());
CREATE POLICY "Users can view their own reports" ON reports FOR SELECT USING (reporter_id = auth.uid());
```

## ⚙️ CODING STANDARDS

### General Rules
1. **Functional components only.** No class components.
2. **Use `'use client'`** directive on any component that uses hooks, event handlers, or browser APIs.
3. **Server components by default.** Only add `'use client'` when necessary.
4. **No `any` types.** Use JSDoc comments for type hints if needed.
5. **Destructure props.** `function Button({ children, onClick })` not `function Button(props)`.
6. **Early returns for guards.** Check auth, loading, error states at the top of components.
7. **One component per file.** Exception: small helper components used only in that file.

### Naming Conventions
- **Files:** `kebab-case.js` for utilities, `PascalCase.js` for components
- **Components:** PascalCase (`ServiceCard`, `BottomNav`)
- **Hooks:** camelCase with `use` prefix (`useAuth`, `useServices`)
- **Constants:** UPPER_SNAKE_CASE (`CATEGORIES`, `BOOKING_STATUSES`)
- **Database columns:** snake_case (`booking_date`, `merchant_id`)
- **CSS classes:** Tailwind utilities only. No custom CSS classes except in globals.css.

### Supabase Patterns
```js
// ALWAYS use the client from lib, never create a new one
import { createClient } from '@/lib/supabase/client';

// ALWAYS handle errors
const { data, error } = await supabase.from('services').select('*');
if (error) {
  console.error('Error fetching services:', error);
  return [];
}

// ALWAYS use select() with specific columns for performance
// BAD:  supabase.from('services').select('*')
// GOOD: supabase.from('services').select('id, title, price, category, merchant:merchants(business_name)')

// ALWAYS use .eq() for filtering, .order() for sorting
// ALWAYS use .limit() to prevent fetching too much data
```

### Component Pattern
```jsx
'use client';

import { useState, useEffect } from 'react';

export default function ServiceCard({ service, onClick }) {
  // 1. Guards / early returns
  if (!service) return null;

  // 2. Hooks
  const [isLoading, setIsLoading] = useState(false);

  // 3. Derived state
  const isBookable = service.status === 'active';

  // 4. Handlers
  const handleClick = () => {
    if (!isBookable) return;
    onClick?.(service);
  };

  // 5. Render
  return (
    <button
      onClick={handleClick}
      className="w-full bg-white border border-nearmee-border rounded-xl p-3 flex gap-3 items-center text-left"
    >
      {/* ... */}
    </button>
  );
}
```

## 🚫 DO NOT DO THESE (common mistakes to avoid)

1. **Do not build a payment system.** No PayMongo, no Stripe, no checkout. Direct pay only.
2. **Do not build an admin dashboard.** Use Supabase dashboard directly for admin tasks.
3. **Do not build push notifications yet.** Simple in-app notification is enough for MVP.
4. **Do not build a chat/messaging system.** Merchants and customers exchange phone numbers.
5. **Do not over-optimize.** No caching layer, no CDN config, no performance tuning until 1000+ users.
6. **Do not build email functionality.** Phone OTP is the only auth method.
7. **Do not create separate merchant app.** One app, mode switch in profile.
8. **Do not use TypeScript.** Plain JavaScript for speed. TypeScript is a Phase 2 upgrade.
9. **Do not add image upload for services yet.** Use emoji/icon per category for MVP.
10. **Do not build subscription payment flow.** Track subscriptions manually via Supabase dashboard + GCash.
